## Vision

A design system that three audiences can consume without translation loss:

- **Designers** work in Figma ‚Äî variables, a sticker sheet, a Token Usage
  table, and component specs.
- **Engineers** consume DTCG-built CSS custom properties, TypeScript tokens,
  and React Aria-based components with Storybook docs.
- **AI agents** read the DTCG JSON (with usage metadata in `$extensions`), the
  knowledge base, Figma‚Üîcode maps, and this site ‚Äî and do most of the
  maintenance via `/ply-*` commands.

## Glossary

- **ADR** ‚Äî Architecture Decision Record. One decision, recorded before
  implementation, never rewritten. See [Decisions](/decisions/).
- **DTCG** ‚Äî the W3C Design Tokens Community Group JSON format
  (`$value`/`$type`/`$description`); our interchange format between Figma and
  code.
- **Primitive / semantic / component tokens** ‚Äî the three tiers: raw values ‚Üí
  meaning ("background/danger") ‚Üí per-component decisions ("button/bg").
- **Sticker sheet** ‚Äî the Figma page that visually displays every token so
  drift is visible to humans.
- **Token Usage table** ‚Äî the Figma table documenting where/how to use each
  token, with do's and don'ts; syncs with variable descriptions.
- **Drift** ‚Äî any disagreement between Figma, the exported tokens, or these
  docs. Caught by `/ply-audit-figma` and `/ply-sync-tokens`, recorded in
  `audits/`.

## 2026-07-05 — Ply gets a face: the Aurora identity

- Three gradient cover concepts (Aurora, Spectrum, Dusk) were designed on the

- Coming from this: branded Figma cover pages for every Ply file, a docs logo
  + favicon, and gradient accents on this site.

## 2026-07-05 — Course correction: reference naming adopted, Ply-first identity

- After maintainer review, a five-front course correction began — identity,

  entry covers the first slice.
- Ply-first identity sweep: the docs site description, hero tagline, README,
  and system-map vision now lead with the name Ply instead of the generic
  "agentic design system" phrasing.
- [ADR-0007](/decisions/adr-0007-adopt-reference-naming/) accepted: Ply's
  token library will mirror the reference library's naming and structure
  verbatim (nested semantic paths, six-rung emphasis ladder, brand + font mode
  collections). Names revert; the 21 contrast value fixes are kept and the
  WCAG contrast lint stays CI-blocking.
- [ADR-0005](/decisions/adr-0005-token-taxonomy/) (token taxonomy) is
  superseded by ADR-0007; `kb/topics/naming.md` notes the reversal.
- Coming next per the approved plan: gradient branding concepts, a
  token-library rename-in-place migration, a real docs build-out (guides +
  token reference + component pages), and a Button rebuild as a single master
  component set.

## 2026-07-05 — Phase 5 begins: Button sets the pattern

- The token library is now a published Figma team library, and the new Core
  Components file consumes it via cross-file variable/style imports — every
  component binding goes through the published tokens.
- [ADR-0006](/decisions/adr-0006-component-anatomy-api/) locked component
  conventions: the variant/tone/size prop trio with closed shared
  vocabularies, exact 1:1 lowercase Figma-property↔code-prop naming (state is
  the documented Figma-only exception), dot-notation composition, and a
  spec-frame template with an accessibility panel.
- Button is designed: three component sets (Button/sm, md, lg) of 32 variants
  each — variant (solid/subtle/ghost) × tone (neutral/accent/danger) × state
  (default/hovered/pressed/disabled). Sizes are split into separate sets to
  stay under Figma's variant-explosion threshold; the size axis maps back to
  one `size` prop in code.
- Every visual property is token-bound: validation found zero unbound fills,
  and heights, paddings, gaps, and radii all bind space/radius tokens. The
  focus ring is documented in the spec frame (2px border/focused outside)
  rather than multiplied through variants — code gets it from React Aria's
  `data-focus-visible`.
- Icons will come from the Phosphor library (enabled in the file), starting
  with IconButton.

## 2026-07-05 — Phase 4: tokens flow from Figma to code — and the contrast lint passes

- The DTCG pipeline is live: the Figma library exports to
  `packages/tokens/src` (primitives plus semantic light/dark files) with
  descriptions, Figma variable IDs, and contrast pairings carried in
  `$extensions.ply`; Style Dictionary builds `dist/css/ply.css` (`:root` +
  `[data-theme="dark"]`, 390 CSS custom properties) and a typed
  `dist/ts/tokens.ts`.
- The contrast lint (`scripts/lint-tokens.mjs`) verifies every declared
  foreground/background pairing against WCAG in both modes, with alpha
  compositing — 180 pair checks now pass.
- The lint's first run found 62 failures and drove real fixes: 21 token values
  changed in Figma (status solids darkened one step in light mode,
  warning/info interactive states now step lighter, dark-mode accent states
  darkened so white text holds, dark secondary text brightened), and two new
  tokens were minted — `text/on-status-warning-bold` and
  `text/on-status-info-bold` — because bold warning/info fills flip light/dark
  across modes. The library is now 349 variables / 162 semantic tokens.
- New machinery: `/ply-export-tokens` (read-only Figma → DTCG → build),
  `/ply-sync-tokens` (drift detection with gated pull/push per the
  [ADR-0005](/decisions/adr-0005-token-taxonomy/) policy),
  `scripts/diff-tokens.mjs` (rename-aware via variable IDs), and a CI workflow
  that builds tokens + docs and fails if the committed dist drifts from src.
- The sticker sheet and the Token Usage table were updated in the same pass
  (labels refreshed, 2 rows added).

## 2026-07-05 — Phase 3 complete: the Token Usage table

- The Token Usage documentation table was built in the Ply Figma file: 160
  rows across 6 tables (Background, Status backgrounds, Text, Icon, Border,
  Data-viz & shadow), each row with a variable-bound sample swatch, the
  token's description, and seeded Where-to-apply / How-to-use / Do / Don't
  guidance.
- Every row carries its variable ID as shared plugin data — rename detection
  and audits key on identity, never on names. Verified: exactly one row per
  semantic token, zero duplicates.
- The `/ply-doc-table` command was added to keep the table in sync:
  idempotent, rename-aware, deprecation-not-deletion, bidirectional
  description sync (table wins usage text per
  [ADR-0005](/decisions/adr-0005-token-taxonomy/)), preview-then-apply.
- This completes Phase 3 (vision step 4): token audit, taxonomy, the rebuilt
  library, styles, sticker sheet, and documentation table are all done. Next:
  Phase 4 — the DTCG export pipeline to code.

## 2026-07-05 — The Ply token library exists in Figma

- The v1 token library was built from scratch in the Ply Figma file under the
  [ADR-0005](/decisions/adr-0005-token-taxonomy/) grammar: 347 variables across
  9 collections — 119 color primitives, 160 semantic tokens (Light + Dark
  modes, 100% alias-based, 100% with descriptions), plus space, radius (spelled
  sizes), font-size/weight/line-height/family, and breakpoints.
- In-file validation passed with zero findings: no duplicate names, no missing
  mode values, no raw hex in the semantic tier, no missing descriptions, and
  the path-prefix invariant holds.
- Six reference-library bugs were fixed during the build rather than ported
  (dark-mode state collisions, a non-monotonic border ramp, an under-contrast
  focus ring, inconsistent dark text steps) — details in
  `audits/ply-build-report-2026-07-05.md`.
- Seven contrast-pairing tokens that had no ancestor in the reference library
  were minted per ADR-0005: `text/on-accent`, `icon/on-accent`,
  `text/on-neutral-bold`, and `text/on-status-*` for all four statuses.
- Still ahead in Phase 3: text styles, effect styles, the Foundations sticker
  sheet, and the Token Usage documentation table.

## 2026-07-05 — Phase 3 begins: baseline audit and the token taxonomy

- Phase 3 strategy changed: the reference token library (a separate Figma file,
  read-only) stays untouched, and Ply's library is built fresh in a new, empty
  Figma file — no rename-in-place migration, no binding-preservation
  constraints.
- Full read-only export of the reference library completed via the Figma MCP:
  10 collections, 395 variables (189 semantic tokens with Light/Dark modes, all
  with descriptions and zero raw hex), 27 text styles, 5 effect styles.
- Baseline audit report written to `audits/ply-baseline-audit-2026-07-05.md`.
  Notable findings: a brand-ramp bug (the 950 step points to the Blue ramp in
  both brand modes), same-value state collisions (e.g. boldest default ≡
  hovered), links bypassing the brand axis, two contrast risks the planned lint
  would catch, and out-of-order insertions in the numbered radius scale.
- [ADR-0005](/decisions/adr-0005-token-taxonomy/) accepted: three tiers;
  element-first properties (`bg`, `text`, `icon`, `border`) with no `color/`
  segment; hyphenated modifier leaves (`bg/accent-hovered`); a 3-rung emphasis
  ladder (subtle/default/bold); the status sub-namespace kept
  (`bg/status/danger`); mandatory `text/on-{role}` contrast pairings with a
  build-time lint; single brand in v1 with the multi-brand axis reserved;
  radius moves to spelled sizes; data-viz and shadow tokens carried forward.
- `kb/topics/naming.md` updated with the adopted-decision note: ADR-0005 is
  canon where the guide disagreed.

## 2026-07-05 — Phase 2: Knowledge base and the design advisor

- The knowledge base launched in `kb/`: 7 system dossiers (Material 3, Polaris,
  Carbon, Primer, Spectrum, Atlassian, shadcn/Radix) researched against live
  2025–2026 sources with a shared 8-section rubric, plus 6 cross-cutting topic
  guides (three-tier tokens, naming, accessibility patterns, theming, component
  API design, agentic design systems) — all with cited sources and retrieval
  dates.
- The design-advisor agent and `/ply-recommend` command are live: proposals are
  checked against the KB and current best practice before adoption, and every
  evaluation is saved to `kb/comparisons/`.
- First real evaluation completed: the token path grammar for the Phase 3
  migration — verdict "adopt modified". It caught that nesting states as path
  children (`bg/accent/hovered` under rest token `bg/accent`) is structurally
  invalid in DTCG, and recommended hyphenated modifier leaves
  (`bg/accent-hovered`), a trimmed 3-rung emphasis ladder, and
  contrast-lint-verified foreground pairings. Dossier:
  `kb/comparisons/2026-07-05-token-path-grammar.md`.
- Notable research findings recorded in the dossiers: Base UI became
  shadcn/ui's default primitive layer (July 2026); Polaris React was deprecated
  for web components (Oct 2025); Spectrum 2 went stable on React Aria
  Components (Dec 2025); Atlassian measured a 52% AI-accuracy gain from their
  agentic layer (DESIGN.md, llms.txt, MCP server).

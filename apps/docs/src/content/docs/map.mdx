## Vision

A design system that three audiences can consume without translation loss:

- **Designers** work in Figma вЂ” variables, a sticker sheet, a Token Usage
  table, and component specs.
- **Engineers** consume DTCG-built CSS custom properties, TypeScript tokens,
  and React Aria-based components with Storybook docs.
- **AI agents** read the DTCG JSON (with usage metadata in `$extensions`), the
  knowledge base, Figmaв†”code maps, and this site вЂ” and do most of the
  maintenance via `/ply-*` commands.

## Glossary

- **ADR** вЂ” Architecture Decision Record. One decision, recorded before
  implementation, never rewritten. See [Decisions](/decisions/).
- **DTCG** вЂ” the W3C Design Tokens Community Group JSON format
  (`$value`/`$type`/`$description`); our interchange format between Figma and
  code.
- **Primitive / semantic / component tokens** вЂ” the three tiers: raw values в†’
  meaning ("background/danger") в†’ per-component decisions ("button/bg").
- **Sticker sheet** вЂ” the Figma page that visually displays every token so
  drift is visible to humans.
- **Token Usage table** вЂ” the Figma table documenting where/how to use each
  token, with do's and don'ts; syncs with variable descriptions.
- **Drift** вЂ” any disagreement between Figma, the exported tokens, or these
  docs. Caught by `/ply-audit-figma` and `/ply-sync-tokens`, recorded in
  `audits/`.

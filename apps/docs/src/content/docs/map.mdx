## Vision

A design system that three audiences can consume without translation loss:

- **Designers** work in Figma — variables, a sticker sheet, a Token Usage
  table, and component specs.
- **Engineers** consume DTCG-built CSS custom properties, TypeScript tokens,
  and React Aria-based components with Storybook docs.
- **AI agents** read the DTCG JSON (with usage metadata in `$extensions`), the
  knowledge base, Figma↔code maps, and this site — and do most of the
  maintenance via `/ply-*` commands.

## Glossary

- **ADR** — Architecture Decision Record. One decision, recorded before
  implementation, never rewritten. See [Decisions](/decisions/).
- **DTCG** — the W3C Design Tokens Community Group JSON format
  (`$value`/`$type`/`$description`); our interchange format between Figma and
  code.
- **Primitive / semantic / component tokens** — the three tiers: raw values →
  meaning ("background/danger") → per-component decisions ("button/bg").
- **Sticker sheet** — the Figma page that visually displays every token so
  drift is visible to humans.
- **Token Usage table** — the Figma table documenting where/how to use each
  token, with do's and don'ts; syncs with variable descriptions.
- **Drift** — any disagreement between Figma, the exported tokens, or these
  docs. Caught by `/ply-audit-figma` and `/ply-sync-tokens`, recorded in
  `audits/`.
